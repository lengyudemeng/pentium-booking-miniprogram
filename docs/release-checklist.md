# 提审前检查清单

更新时间：2026-04-08

> 以下清单结合本项目当前实现整理；其中微信小程序隐私相关检查点，参考了微信开放文档/官方示例入口与后台提审流程说明。

---

## A. 隐私合规

### 必查

- [ ] 微信公众平台后台已配置 **《用户隐私保护指引》**
- [ ] 提审页面已正确选择“**采集用户隐私**”
- [ ] 指引中已声明本项目实际处理的信息：
  - [ ] 姓名
  - [ ] 手机号
  - [ ] 预约需求
- [ ] 指引中已填写：
  - [ ] 收集目的
  - [ ] 使用场景
  - [ ] 保存期限
  - [ ] 删除方式
  - [ ] 联系方式
- [ ] 小程序内可查看《隐私政策》
- [ ] 小程序内可查看《用户协议》
- [ ] 预约提交前必须勾选同意《隐私政策》《用户协议》
- [ ] `miniprogram/utils/legal.js` 中联系方式不是占位文案

### 建议复核

- [ ] 页面文案与后台《用户隐私保护指引》一致
- [ ] 数据保存期限与实际数据库清理策略一致
- [ ] 删除方式真实可执行
- [ ] 联系方式真实可用

---

## B. 权限与安全

### 管理员

- [ ] `settings/global.adminOpenIds` 已配置正式管理员 OpenID
- [ ] 至少配置 2 个管理员 OpenID，防止单点故障
- [ ] 普通账号看不到管理员入口
- [ ] 非白名单用户手动访问后台时会被拦截

### 服务人员

- [ ] 服务人员权限必须走申请审批流程
- [ ] 服务人员分组只能由管理员分配
- [ ] 不存在默认绑定码 / 默认密码
- [ ] 代码中不再保留：
  - [ ] `DEFAULT_BIND_CODE`
  - [ ] `DEFAULT_ADMIN_PASSWORD`
  - [ ] 管理员密码校验接口

### 数据安全

- [ ] 数据库权限规则已检查
- [ ] 敏感字段不允许前端直改
- [ ] 云函数已重新上传部署

---

## C. 功能验收

### 预约流程

- [ ] 用户可正常选择日期、时段、预约类型
- [ ] 用户填写姓名/手机号/需求后可正常提交
- [ ] 未勾选协议时不能提交
- [ ] 同一用户同一天不能重复提交
- [ ] 当天及之后不可取消预约

### 服务人员流程

- [ ] 普通用户可提交服务人员申请
- [ ] 管理员可审批通过/驳回
- [ ] 审批通过后服务人员可看到所属分组预约
- [ ] 服务人员仅能结单自己组别的预约

### 管理员流程

- [ ] 管理员可进入后台
- [ ] 管理员可审批服务人员
- [ ] 管理员可修改服务人员分组
- [ ] 管理员可保存学期轮值配置
- [ ] 管理员可关闭/开启某日预约

---

## D. 提审展示与文案

- [ ] 首页文案无乱码
- [ ] 预约页文案无乱码
- [ ] 我的页文案无乱码
- [ ] 管理页文案无乱码
- [ ] 页面标题与实际功能一致
- [ ] 无“测试中”“占位图”“默认密码”等敏感演示信息

---

## E. 部署检查

- [ ] `miniprogram/utils/config.js` 已填正式云环境 ID
- [ ] 云函数 `bookingService` 已上传并部署
- [ ] 数据库集合已创建
- [ ] `settings/global` 已存在
- [ ] 线上环境数据与测试环境数据已区分

---

## F. 建议补充的人工核验

- [ ] 用普通用户账号完整走一遍预约流程
- [ ] 用待审批账号完整走一遍申请流程
- [ ] 用管理员账号完整走一遍审批和分组流程
- [ ] 用服务人员账号完整走一遍查看预约和结单流程
- [ ] 真机预览检查页面排版、按钮点击、下拉刷新

- [ ] 有隐私政策页面
- [ ] 有用户协议页面
- [ ] 有协议勾选
- [ ] 有管理员白名单
- [ ] 有服务人员审批
- [ ] 有真实联系方式
- [ ] 已上传最新云函数
- [ ] 已完成真机自测
